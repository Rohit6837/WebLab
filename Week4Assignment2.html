Version Control

Version control allows you to keep track of your work and helps you to easily explore the changes you have made in your projects.
With version control software such as Git, version control is much smoother and easier to implement on an online platform like Github.
Having a GitHub repo makes it easy for you to keep track of collaborative and personal projects since each file on GitHub has a history
making it easier to explore the changes that occurred to it at different points of time. For personal projects, version control allows you to keep
track of your work and provides an easy navigation among the many versions of the files you create while maintaining an online backup too.

GitHub Commands

Clone

  • Use git clone command to create a copy of a specific repository

  • By cloning we get all files , branches , commits of the repo

  • After a repo is cloned we can interact with it localy

Push

  • Use git push to push commits made on the local branch to a remote repo.

  • It takes first argument as a remote name for example origin.

  • Branch name for example master is given as second argument.

Commit

  • Use git commit command to save changes to the local repo.

  • We can also give a commit message using -m

  • Only the changes added in staging area are commited.
